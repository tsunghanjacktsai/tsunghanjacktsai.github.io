169. Majority Element

Source link: https://leetcode.com/problems/majority-element/

Design

Solution I

The hash table could be applied to record the number of each element.

  1. Initialize a hash map to be a table for recording.
  2. Check if nums[i] exists inside the array nums, if so, increase the value of the element, else assign 1 for the key nums[i].
  3. If the value of the element nums[i] larger than (n / 2), return nums[i].

Solution II

The problem could be solved by the Boyer-Moore Voting Algorithm, the basic idea of such algorithm is remove elements having the same values, only the majority one will be last inside the array.

  1. Initialize the variable res for recording the result and variable cnt for the existing times of each element.
  2. Traverse through the array nums, increase the cnt if the result is equal to nums[i], else if res not equal to nums[i] and cnt is larger than 0, decrease the value of cnt (remove the current value), else set the value of res to nums[i] and set the cnt back to 1.

Implementation

Solution I

class Solution {
  public int majorityElement(int[] nums) {
    HashMap<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
      if (map.containsKey(nums[i])) {
        map.put(nums[i], map.get(nums[i]) + 1);
      } else {
        map.put(nums[i], 1);
      }
      if (map.get(nums[i]) > nums.length / 2) {
        return nums[i];
      }
    }
    return -1;
  }
}

Solution II

class Solution {
  public int majorityElement(int[] nums) {
    int res = nums[0], cnt = 1;
    for (int i = 1; i < nums.length; i++) {
      if (res == nums[i]) {
        cnt++;
      } else if (res != nums[i] && cnt > 0) {
        cnt--;
      } else {
        res = nums[i];
        cnt++;
      }
    }
    return res;
  }
}

Complexity Analysis

Solution I

Time: $O(n)$

Space: $O(n)$

Solution II

Time: $O(nlogn)$

Space: $O(n)$
