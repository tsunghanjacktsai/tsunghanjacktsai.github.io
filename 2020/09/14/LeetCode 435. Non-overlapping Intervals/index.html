LeetCode 435. Non-overlapping Intervals

Problem

Source link: Non-overlapping Intervals

Given a collection of intervals, find the minimum number of intervals you need to remove to make the rest of the intervals non-overlapping.

Example 1:

Input: [[1,2],[2,3],[3,4],[1,3]]
Output: 1
Explanation: [1,3] can be removed and the rest of intervals are non-overlapping.

Example 2:

Input: [[1,2],[1,2],[1,2]]
Output: 2
Explanation: You need to remove two [1,2] to make the rest of intervals non-overlapping.

Example 3:

Input: [[1,2],[2,3]]
Output: 0
Explanation: You don't need to remove any of the intervals since they're already non-overlapping.

Note:

  1. You may assume the interval’s end point is always bigger than its start point.
  2. Intervals like [1,2] and [2,3] have borders “touching” but they don’t overlap each other.

Design

The most complex part of this problem is to determine which interval to preserve while an overlap exists. As we did not know how large the starting point of the next interval will be, we always need to select the smaller ending point between overlapping intervals to create more spaces for the next interval. So the main process is described below:

  1. Sort intervals by their starting points (ascending).
  2. Record ending point of the previous selected interval and number of non-overlapping intervals for the result.
  3. If the starting point of a current interval is smaller than the ending point of its previous one, which means an overlap exists, increase the number of non-overlapping intervals. Otherwise, select an interval which has the smaller ending point. Don’t forget to record the previous ending point as well.
  4. Return the result by subtracting the number of non-overlapping intervals from the array length.

Implementation

class Solution {
  public int eraseOverlapIntervals(int[][] intervals) {
    if (intervals == null || intervals.length < 2) return 0;

    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    int nonOverlapNum = 0;
    int prevEnd = Integer.MIN_VALUE;

    for (int[] interval : intervals) {
      if (interval[0] >= prevEnd) {
        nonOverlapNum++;
        prevEnd = interval[1];
      } else {
        prevEnd = Math.min(prevEnd, interval[1]);
      }
    }

    return intervals.length - nonOverlapNum;
  }
}

Complexity Analysis

Time: $O(nlogn)$

Space: $O(n)$
