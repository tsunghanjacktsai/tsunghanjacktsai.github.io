LeetCode 452. Minimum Number of Arrows to Burst Balloons

Problem

Source link: Minimum Number of Arrows to Burst Balloons

There are a number of spherical balloons spread in two-dimensional space. For each balloon, provided input is the start and end coordinates of the horizontal diameter. Since it’s horizontal, y-coordinates don’t matter and hence the x-coordinates of start and end of the diameter suffice. Start is always smaller than end. There will be at most 104 balloons.

An arrow can be shot up exactly vertically from different points along the x-axis. A balloon with xstart and xend bursts by an arrow shot at x if xstart ≤ x ≤ xend. There is no limit to the number of arrows that can be shot. An arrow once shot keeps travelling up infinitely. The problem is to find the minimum number of arrows that must be shot to burst all balloons.

Example:

Input:
[[10,16], [2,8], [1,6], [7,12]]

Output:
2

Explanation:
One way is to shoot one arrow for example at x = 6 (bursting the balloons [2,8] and [1,6]) and another arrow at x = 11 (bursting the other two balloons).

Design

One of the key points of this problem is to realize that simply merging intervals cannot get what we want. Take the following input for example:

[[10,16], [2,8], [1,6], [7,12]]

If we simply merge those intervals, we will get [1, 16] as the result. However, by understanding the purpose of this problem, the correct answer is 2, as we cannot burst all balloons which [1, 6] and [7, 12] does not overlap.

The other key point is to determine how to sort those intervals. As common intervals related problems, we often try sorting by their starting points first. However, for this problem, if we choose starting points as the sorting basis, we need to calculate the minimum ending point among previous overlapping intervals every time when an interval overlaps with the previous interval which has the minimum ending point. This can be simplified if we sort intervals by their ending points, which the process of calculating the minimum ending point of previous intervals can be omitted.

Let’s see how the main process is:

  1. Sort points intervals first (by starting or ending points).
  2. Start traversing intervals, increase the arrow number once if a current starting point is larger than the minimum ending point of previous intervals.

The section below will show how both ways of sorting affect the calculation process.

Implementation

Sorting by Starting points

class Solution {
  public int findMinArrowShots(int[][] points) {
    if (points == null || points.length == 0) return 0;

    Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));

    int minArrow = 1;
    int prevEnd = points[0][1];

    for (int i = 1; i < points.length; i++) {
      if (points[i][0] > prevEnd) {
        minArrow++;
        prevEnd = points[i][1];
      } else {
        prevEnd = Math.min(prevEnd, points[i][1]);
      }
    }

    return minArrow;
  }
}

Sorting by Ending Points

class Solution {
  public int findMinArrowShots(int[][] points) {
    if (points == null || points.length == 0) return 0;

    Arrays.sort(points, (a, b) -> Integer.compare(a[1], b[1]));

    int minArrow = 1;
    int prevEnd = points[0][1];

    for (int i = 1; i < points.length; i++) {
      if (points[i][0] > prevEnd) {
        minArrow++;
        prevEnd = points[i][1];
      }
    }

    return minArrow;
  }
}

Complexity Analysis

Time: $O(nlogn)$

Space: $O(n)$
