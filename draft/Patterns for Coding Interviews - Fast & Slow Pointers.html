Approach

The fast & slow pointers pattern can solved problems by moving two pointers with different speeds. The method is quite useful when dealing with situations containing cyclic linked lists or arrays. As long as lists or arrays have a cycle, the fast pointer (i.e. the pointer travel more steps at a time than the other one) will definitely meet the slow pointer in the end.

Here’s a classic problem Linked List Cycle, which requires us to check if there is a cycle in a linked list. Some examples are shown below:

Input: head = [3,2,0,-4], pos = 1
Output: true
Explanation: There is a cycle in the linked list, where the tail connects to the 1st node (0-indexed).
Input: head = [1,2], pos = 0
Output: true
Explanation: There is a cycle in the linked list, where the tail connects to the 0th node.

Input: head = [1], pos = -1
Output: false
Explanation: There is no cycle in the linked list.

The solving process is demonstrated in the image below, which the slow pointer is denoted by S and the fast pointer is denoted by F.

As the figure shown, the fast pointer moves two steps instead of one step at a time as the slow pointer (i.e. 2x speed than the slow pointer). They will meet in the end if a cycle exists. Let’s see the implementation below:

/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */
public class Solution {
  public boolean hasCycle(ListNode head) {
    ListNode dummy = new ListNode(0), slow = dummy, fast = dummy;
    dummy.next = head;
    while (fast != null && fast.next != null) {
      slow = slow.next;
      fast = fast.next.next;
      if (slow == fast) {
        return true;
      }
    }
    return false;
  }
}

Let’s try again with the variation of this problem (Linked List Cycle II). Find out the starting point of a cycle if there is. The main process is shown below:

As you can see, we will check if there is a cycle exists first, which the method is the same as the last problem. Next, relocate the slow pointer back to the head of the list, and start moving forward 1 step at a time with the fast pointer. Once if they meet, we can find out where the starting point of the cycle is. The code is implemented below:

/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */
public class Solution {
  public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    boolean isCycle = false;
    while (fast != null && fast.next != null) {
      slow = slow.next;
      fast = fast.next.next;
      if (slow == fast) {
        isCycle = true;
        break;
      }
    }
    if (!isCycle) return null;
    slow = head;
    while (slow != fast) {
      slow = slow.next;
      fast = fast.next;
    }
    return slow;
  }
}

Conclusion

Now we have seen two typical problems that could be solved by the fast & slow pointers approach. There are still some similar problems such as finding out the nth node from the end of a list or checking if the palindrome exists within a list. While applying the fast & slow pointers approach, we always need to consider:

  1. The difference of speeds between slow & fast pointers. Do we need to change their speeds to get the result?
  2. What happens if the slow & fast pointers meet or not meet in the end?
