Patterns for Coding Interviews - Cyclic Sort

Problem

  1. First Missing Positive
  2. Missing Number
  3. Find the Duplicate Number
  4. Find All Duplicates in an Array
  5. Find All Numbers Disappeared in an Array

Approach

The cyclic sort pattern is mainly focused on solving problems that required us to find something such as duplicates or missing numbers in an unsorted array within $O(n)$ time and constant space. Some of them will include a typical characteristic which all elements inside the array are positive, i.e. $1 \leq nums[i] \leq n$ (n = size of array). Let’s find the feeling through some examples.

We will first go on with the easy Leetcode problem Missing Number. As the problem stated, an array containing distinct integers from 0 to n has one missing number and need to find out what it is. Few cases are provided below:

Input: [3,0,1]
Output: 2
Input: [9,6,4,2,3,5,7,0,1]
Output: 8

By observing the problem, we could find out a potential condition from the description that elements within the array are from 0 to n and are distinct. It means that all elements are positive except for the element 0 and exist only once. Thus, simply speaking, we can sort the array and find out the missing number by 0 as the description below:

  1. If 0 exists, we can use 0 as a point to mark the missing positive out.
  2. If 0 does not exist, which means that all other elements are at the right place, we can just simply return 0.

Now, we can get back to the main part: How exactly can we solve the problem in constant space and $O(n)$ time? The key point is the sorting function. You may probably know that common sorting algorithms without extra space will result in $O(nlogn)$ time in the worst case, which does not satisfy what this problem asked. So we need to do some special treatments. As we already know that the array only contains 0 as well as distinct positives, we could try to replace those positives according to their values, that is, place an element 1 to the index 0 and an element 9 to the index 8, which nums[i] will be swapped with nums[nums[i] - 1]. The detailed process is shown in the image below:

Now, let’s see how the solution be implemented:

class Solution {
  public int missingNumber(int[] nums) {
    int i = 0;
    while (i < nums.length) {
      if (nums[i] > 0 && nums[i] != nums[nums[i] - 1]) {
        swap(nums, i, nums[i] - 1);
      } else {
        i++;
      }
    }
    for (i = 0; i < nums.length; i++) {
      if (nums[i] == 0) return i + 1;
    }
    return 0;
  }

  private void swap(int[] nums, int i, int j) {
    int tmp = nums[i];
    nums[i] = nums[j];
    nums[j] = tmp;
  }
}

Of course, this is not the optimal way to solve this problem. We can use some math methods to enhance the efficiency. However, that is unrelated to the cyclic sort pattern, so we will just skip this part.

Are you feeling it now? If not, let’s try to solve a similar problem Find All Duplicates in an Array. Same as the last one, we can find the constraint $1 \leq nums[i] \leq$ from the problem statement, which means that we can apply the cyclic sort pattern to complete the sort function. The only difference is, we need to get multiple elements for the result.

Last time, we can utilize 0, i.e. the only non-positive integer, as a mark for the missing number. To solve this problem, we need to mark those duplicates out by checking whether all elements are at the right place in the end, e.g. an element at the index 0 should be equal to 1 and an element at the index 3 should be equal to 4. So we can conclude that nums[i] is equal to (i + 1). Duplicates exist while nums[i] is not equal to (i + 1) and should be put into the result list. Look at the image below for details:

Here is the code implementation:

class Solution {
  public List<Integer> findDuplicates(int[] nums) {
    int i = 0;
    while (i < nums.length) {
      if (nums[i] != nums[nums[i] - 1]) {
        swap(nums, i, nums[i] - 1);
      } else {
        i++;
      }
    }
    List<Integer> result = new LinkedList<>();
    for (i = 0; i < nums.length; i++) {
      if (nums[i] != (i + 1)) {
        result.add(nums[i]);
      }
    }
    return result;
  }

  private void swap(int[] nums, int i, int j) {
    int tmp = nums[i];
    nums[i] = nums[j];
    nums[j] = tmp;
  }
}

Conclusion

So far, we have seen two problems that can be solved by cyclic sort pattern. Most of the array in this kind of problems could be sorted through the same method varies with small differences. After, find our target in the sorted array by problem’s conditions. In conclusion, the main solving process can be separate into two parts:

  1. Sort the array by placing elements with corresponding indices based on their values.
  2. Obtain the result by checking if all elements are at the correct places.
