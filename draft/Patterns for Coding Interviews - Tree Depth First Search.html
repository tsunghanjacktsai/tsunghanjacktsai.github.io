Approach

The tree DFS pattern is an algorithm that apply DFS technique while searching within a tree. The feature of this pattern is to traverse each tree branch as far as possible before backtracking. It can be implemented in the iterative or recursive ways. This article will be mainly focus on discussing the recursive method.

Before we start, let’s recall three tree traversal methods, which are pre-order, in-order and post-order. The templates of them are presented below:

Pre-order traversal:

void preorder(TreeNode root) {
  resuld.add(root);
  preorder(root.left);
  preorder(root.right);
}

In-order traversal:

void inorder(TreeNode root) {
  inorder(root.left);
  result.add(root);
  inorder(root.right);
}

Post-order traversal:

void postorder(TreeNode root) {
  postorder(root.left);
  postorder(root.right);
  result.add(root);
}

Keep those three in mind, which most tree DFS problems can be solved by either of them.

Now let’s get started. We will first look at the Leetcode easy problem Path Sum as a warm-up. The main target of this problem is to see if there is a path (root to leaf) which its sum is equal to the given number. An example is provided below:

Example:

Given the below binary tree and sum = 22,

      5
     / \
    4   8
   /   / \
  11  13  4
 /  \      \
7    2      1

return true, as there exist a root-to-leaf path 5->4->11->2 which sum is 22.

As we mentioned before, most tree DFS problems can be solved by in-order, pre-order, or post-order traversal. Hence the first step in solving this problem is to decide which kind of traversal we should apply.

It is quite obvious that the in-order traversal is a more suitable way as a path contains nodes starting from root to leaf, which is a top-down traversal. Therefore, the algorithm process will be like:

  1. See if the current node is a leaf and whether the accumulated sum, i.e. path sum, is equal to our target.
  2. Recursively calculating the path sum of left subtrees and right subtrees.

Here’s the implementation:

/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
  public boolean hasPathSum(TreeNode root, int sum) {
    if (root == null) {
      return false;
    }

    if (root.val == sum && root.left == null && root.right == null) {
      return true;
    }

    return hasPathSum(root.left, sum - root.val) || hasPathSum(root.right, sum - root.val);
  }
}

Have you seen the pattern? Maybe only one problem is not enough. Let’s try an advanced one — Validate Binary Search Tree. This problem required us to determine if a binary tree is a valid BST. See some examples below:

Example 1:

    2
   / \
  1   3

Input: [2,1,3]
Output: true

Example 2:

    5
   / \
  1   4
     / \
    3   6

Input: [5,1,4,null,null,3,6]
Output: false
Explanation: The root node's value is 5 but its right child's value is 4.

Think about how you would do this inside your brain. I believe most people would first take the root node as a reference and compare it with nodes from left and right subtrees.

Always remember the first step of solving tree DFS problems is always to decide which kind of traversals you would like to apply. The thought mentioned above is clearly the in-order traversal (root → left → right) method.

Now we can dive into the main part this algorithm. Be careful with the definition of BST:

Assume a BST is defined as follows:

  • The left subtree of a node contains only nodes with keys less than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • Both the left and right subtrees must also be binary search trees.

If you only compare the node with its left and right child instead of left and right subtrees, the case shown below will return the wrong answer:

    5
   / \
  3   8
 / \ 
2   6

Input: [5,3,8,2,5,null,null]
Output: true
Explanation: 6 is larger than the root node's value, but we only compare it with 3.

To avoid the situation described above, we should record the root node’s value to enable comparing from it during each traversal. Hence the implementation looks like below:

/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
  public boolean isValidBST(TreeNode root) {
    return isValidBST(root, null, null);
  }

  private boolean isValidBST(TreeNode root, TreeNode min, TreeNode max) {
    if (root == null) {
      return true;
    }
    if ((min != null && root.val <= min.val) || (max != null && root.val >= max.val)) {
      return false;
    }
    return isValidBST(root.left, min, root) && isValidBST(root.right, root, max);
  }
}

Conclusion

In summary, the process of tree DFS technique (recursive) can be divided into steps below:

  1. Determine which kind of tree traversal method is more suitable to solve the problem.
  2. Find the base case for the recursive process (e.g. root == null).
  3. Follow the determined traversal method to define main (root) and subproblems (left and right subtrees) and how we can solve them.
