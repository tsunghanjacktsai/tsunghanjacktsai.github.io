Approach

In general, the sliding window pattern is focus on dealing with problems asking us to find and calculate something among contiguous subarrays of a given size array. The basic template of this pattern looks like below:

int windowStart = 0 windowEnd = 0;

while (windowEnd < s.size()) &#123;
  char rightChar = s[windowEnd];

  // TODO: Do some updates of the window

  windowEnd++;

  while (window needs shrink) &#123;
    char leftChar = s[windowStart];

    // TODO: Do some updates of the window

    windowStart++;
  &#125;
&#125;

Take Minimum Window Substring for example, given an array of n positive integers and a positive integer s, find the minimal length of a contiguous subarray of which the sum >= s, if there isn’t one, return 0 instead. Some cases are shown below:

Input: s = 7, nums = [2,3,1,2,4,3]
Output: 2
Explanation: the subarray [4,3] has the minimal length under the problem constraint.
Input: s = 100, nums = []
Output: 0
Explanation: the subarray is empty so that there is no sum within it larger than s.

The brute-force way of solving this problem is to simply travel through each element and check if its sum with previous elements until the sum is equal or larger than s. After, compare the current length of the subarray with the minimum length recorded so far. This kind of approach will take $O(n^2)$ time complexity.

To be more efficient, the sliding window approach can be applied as the problem is asking us to calculate the sum of elements inside a contiguous subarray. As the window sliding ahead, the next element will be merged into it for obtaining the sum of the next subarray. Once it has been equal or larger than s, compare the length of the window with the current minimal length. Meanwhile, start to shrink the window, i.e., substract the element moving out of the window until the sum of elements within the window has been smaller than s. The process will be like:

  1. Merge the next element into the window and add its value with the sum of those elements.
  2. Once the sum of elements inside the window has been equal or greater than s, update the minimal length if the size of window is smaller. In the meantime, substract the element moving out of the window.

The algorithm will be looked like:

class Solution &#123;
  public int minSubArrayLen(int s, int[] nums) &#123;
    int windowStart = 0, windowEnd = 0, windowSum = 0, minLen = Integer.MAX_VALUE;

    while (windowEnd < nums.length) &#123;
      windowSum += nums[windowEnd++];

      while (windowSum >= s) &#123;
        minLen = Math.min(minLen, windowEnd - windowStart);
        windowSum -= nums[windowStart++];
      &#125;
    &#125;

    return minLen == Integer.MAX_VALUE ? 0 : minLen;
  &#125;
&#125;

Conclusion

As a consequence, we only need to think about four questions while we trying to apply the sliding window pattern, which are:

  1. While extending the end of the window, i.e., merging the new element, which data should be updated?
  2. In what condition should the window be stopped extending, and start to shrink the window?
  3. While moving the start of the window, i.e., substracting the element, which data should be updated?
  4. When the data we required should be updated? (extending/shrinking the window)
