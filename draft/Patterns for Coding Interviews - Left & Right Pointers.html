Approach

The left & right pointers pattern could be used to solve problems which require us to find a set of elements within an array or list that meet certain conditions. The basic form of this approach is doing the binary search to find the target within a sorted array:

int left = 0, right = nums.length - 1;
while (left <= right) {
  int mid = left + (right - left) / 2;
  if (nums[mid] == target) {
    return mid;
  } else if (nums[mid] > target) {
    right = mid - 1;
  } else {
    left = mid + 1;
  }
}

Take 3Sum for example, find all unique triplets containing a, b and c within the array nums which a + b + c = 0. Some cases are shown below:

Input: nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
Input: nums = []
Output: []
Input: nums = [0]
Output: []

The brute force approach of solving this problem is to traverse through the array and try each triplet that has been met. It will cost $O(n^3)$ for the time complexity, and $O(1)$ for the space complexity (ignoring the space required for the output array).

The two pointers (left & right) pointers approach can be applied to accelerate the performance. To obtain a clearer observation, replace the formula a + b + c = 0 by a + b = -c. The sum of elements where a and b at should be equal to negative c. The main process of the algorithm is shown as below:

  1. Sort the array nums, which can facilitate the calculation of the two pointers approach (similar to the binary search).
  2. Iterate through the array nums, set up the first element and start searching last two elements of each triplet by left and right pointers.
  3. Add nums[i] with elements where left and right pointers at. If the sum is equal to 0, put the triplet into the result array. Then move the left pointer forward and the right pointer backward. There’s one thing should be noticed, which we cannot return the triplet with the same elements (values) as others (unique triplet).
  4. If the sum of nums[i] and elements where left and right pointers at is smaller than 0, move the left pointer forward. On the other hand, move the right pointer backward if the sum is larger than 0.

The implementation will be looked like:

class Solution {
  public List<List<Integer>> threeSum(int[] nums) {
    List<List<Integer>> res = new ArrayList<>();
    Arrays.sort(nums);
    for (int i = 0; i < nums.length - 2; i++) {
      if (i > 0 && nums[i] == nums[i - 1]) {
        continue;
      }
      int left = i + 1, right = nums.length - 1;
      while (left < right) {
        int sum = nums[i] + nums[left] + nums[right];
        if (sum == 0) {
          res.add(Arrays.asList(nums[i], nums[left], nums[right]));
          left++;
          right--;
          while (left < right && nums[left] == nums[left - 1]) {
            left++;
          }
          while (left < right && nums[right] == nums[right + 1]) {
            right--;
          }
        } else if (sum < 0) {
          left++;
        } else {
          right--;
        }
      }
    }
    return res;
  }
}

The time complexity while applying left & right pointers approach can be reduced to $O(n ^ 2)$, and the space complexity will increase to $O(n)$ required by sorting function.

Conclusion

There are several variations of this pattern. We could come up with this method while solving problems required for searching within an array, especially for the sorted set. Some conditions should be considered while applying this approach:

  1. If there is any situation which left pointer and right pointer should be moved simultaneously.
  2. If there are situations that only need to move either left or right pointer.
  3. If there is any return condition required in the loop.
