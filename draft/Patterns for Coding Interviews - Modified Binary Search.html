Approach

Being asked to search for elements within a given sorted list or array is very common in technical interviews. One of the best solutions to handle this kind of problems is binary search pattern.

You might think that binary search is so simple that you can even write it with your eyes closed. However, binary search approach could be varied in different kinds of situations. People would often fail on dealing with its edge cases or conditions.

A few common mistakes should be careful here:

  1. The break condition of the search loop can be varied in different problems. But if you place start < end as the condition, you will probably miss the target element while it is at the place where the left or right pointer is.

    We use start <= end if we are returning the match from inside the loop. We use start < end when we want to exit the loop first and then use the result of lo or hi to return the match.

  2. Someone would get the middle point by (start + end) / 2. It is an intuitive way to write like this, however, you might get overflow error when the value startand end is big. A great way to deal with this is start + (end - start), or even more efficient waystart + (end - start) >>> 1.

  3. The update of left and right pointers can also be varied under different scenes. The easiest way to determine their movement is to see if the current middle element is still possible to become the result after an iteration.

First, we look at an easy LeetCode problem Binary Search, which requires us to find a target element in within a sorted integer array. Here are some examples:

Example:

Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4
Explanation: 9 exists in nums and its index is 4
Input: nums = [-1,0,3,5,9,12], target = 2
Output: -1
Explanation: 2 does not exist in nums so return -1

This can be easily solved by the most basic form of binary search. The idea is as follows:

  1. Define a start pointer and an end pointer which pointing to the start and end element of the array respectively.
  2. Find out the middle point of these two pointers.
  3. If the middle element is equal to the target value, return its index. Otherwise, move the start or end pointer to narrow down the search scope.
  4. Repeat from the second step until we find the target or else return -1 as the result.

The visual explanation and code implementation are shown below:

class Solution {
  public int search(int[] nums, int target) {
    if (nums == null || nums.length == 0) return -1;

    int start = 0, end = nums.length - 1;
    while (start <= end) {
      int mid = start + (end - start) / 2;

      if (target > nums[mid]) {
        start = mid + 1;
      } else if (target < nums[mid]) {
        end = mid - 1;
      } else {
        return mid;
      }
    }

    return -1;
  }
}

The time complexity of basic binary search as above is $O(logn)$, and the space complexity is $O(1)$.

After learning the basic binary search, we will then go on an advanced one. Let’s try a LeetCode Medium problem Find Peak Element. The problem requires us to find the peak element of an array. Here comes a few examples:

Example 1:

Input: nums = [1,2,3,1]
Output: 2
Explanation: 3 is a peak element and your function should return the index number 2.

Example 2:

Input: nums = [1,2,1,3,5,6,4]
Output: 1 or 5 
Explanation: Your function can return either index number 1 where the peak element is 2, 
             or index number 5 where the peak element is 6.

This one is a bit different, which you are not asked to search for a given value.

First, think about how to identify a peak element within a set of integers. By observing the examples above, we can easily find that the only difference between the peak element and others is that its value is greater than the next element. So the algorithm process will be like below:

  1. Find out the middle element of the array.
  2. Compare the middle point with its next element.
  3. Check if the middle point is larger than its next element. If so, the peak element is on the left side of the array. Here we should be careful that the middle point is still possible to be the answer, so we only discard its right part, i.e. end = mid. Otherwise, search on the right side of the middle point.
  4. Go back to the second step.
  5. Since we are storing the peak element with the start pointer. We should end the loop when start = end and return the value of the start pointer.

class Solution {
  public int findPeakElement(int[] nums) {
    int start = 0, end = nums.length - 1;
    while (start < end) {
      int mid = start + (end - start) / 2;

      if (nums[mid] > nums[mid + 1]) {
        end = mid;
      } else {
        start = mid + 1;
      }
    }

    return start;
  }
}

Conclusion

In summary, we should always focus on these three things while solving binary search problems:

  1. How to get the middle point?
  2. How to move the left and right pointers?
  3. What is the break condition of the search loop?
