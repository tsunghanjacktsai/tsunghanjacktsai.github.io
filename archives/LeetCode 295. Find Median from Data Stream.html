LeetCode 295. Find Median from Data Stream

Problem

Source link: Find Median from Data Stream

Median is the middle value in an ordered integer list. If the size of the list is even, there is no middle value. So the median is the mean of the two middle value.

For example,

[2,3,4], the median is 3
[2,3], the median is (2 + 3) / 2 = 2.5

Design a data structure that supports the following two operations:

  • void addNum(int num) - Add a integer number from the data stream to the data structure.
  • double findMedian() - Return the median of all elements so far.

Example:

addNum(1)
addNum(2)
findMedian() -> 1.5
addNum(3) 
findMedian() -> 2

Design

The brute-force way of solving this problem is to simply insert all given numbers into an ordered list like insertion sort, which costs $O(n)$ time. So we can return the median immediately. That doesn’t sound bad, right? But is there a way that we could improve it?

The answer is yes.

The two heap pattern introduced here is able to make some improvement. As the median is the middle value of a set of numbers, the main idea here is to separate the list into two halves, which a half is smaller than the median and the other half is larger. So we could save the smaller half into a minimum heap and the larger half into a maximum heap. If the number of elements is odd, the median will be the peak element of either maximum or minimum heap. Otherwise, it will be the average of peak elements in both heaps.

The most essential thing here is that we need to balance the sizes of both heaps. If the maximum heap size becomes larger than the minimum heap size + 1, move the peak element of the maximum heap into the minimum heap. Otherwise, do it the same while the minimum heap size is larger than the maximum heap size.

Implementation

class MedianFinder &#123;

  PriorityQueue<Integer> maxHeap;
  PriorityQueue<Integer> minHeap;

  /** initialize your data structure here. */
  public MedianFinder() &#123;
    maxHeap = new PriorityQueue<>((a, b) -> b - a);
    minHeap = new PriorityQueue<>((a, b) -> a - b);
  &#125;

  public void addNum(int num) &#123;
    if (maxHeap.isEmpty() || maxHeap.peek() >= num) &#123;
      maxHeap.offer(num);
    &#125; else &#123;
      minHeap.offer(num);
    &#125;

    if (maxHeap.size() > minHeap.size() + 1) &#123;
      minHeap.offer(maxHeap.poll());
    &#125; else if (maxHeap.size() < minHeap.size()) &#123;
      maxHeap.offer(minHeap.poll());
    &#125;
  &#125;

  public double findMedian() &#123;
    if (maxHeap.size() == minHeap.size()) &#123;
      return maxHeap.peek() / 2.0 + minHeap.peek() / 2.0;
    &#125;
    return maxHeap.peek();
  &#125;
&#125;

Complexity

Time: $O(logn)$ (insertNum), $O(1)$ (findMedian)

Space: $O(n)$
