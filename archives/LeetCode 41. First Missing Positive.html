LeetCode 41. First Missing Positive

Problem

Source link: First Missing Positive

Given an unsorted integer array, find the smallest missing positive integer.

Example 1:

Input: [1,2,0]
Output: 3

Example 2:

Input: [3,4,-1,1]
Output: 2

Example 3:

Input: [7,8,9,11,12]
Output: 1

Follow up:

Your algorithm should run in O(n) time and uses constant extra space.

Design

The problem requires us to provide a solution with O(n) time and constant space, which is a typical problem that should be solved by the cyclic sort pattern. The main process of the algorithm is described below:

  1. Start traversing through the array.
    1. Ignore those elements that are negative or larger than the array length.
    2. Otherwise, swap nums[i] to the correct place (2 to index 1).
  2. Check if all elements are at where they should be, e.g. 1 at index 0, 3 at index 2. If nums[i] is not equal to i + 1, which means that we have found the missing positive, return i + 1 as an answer.
  3. Be careful for some edge cases like [1, 2, 3] (all elements are at where they should be) or an empty array. For those examples, [1, 2, 3] should return 4 and the empty array should return 1, which can be sum up that those kinds of situations will return the array length + 1.

Implementation

class Solution {
  public int firstMissingPositive(int[] nums) {
    int i = 0;
    while (i < nums.length) {
      if (nums[i] > 0 && nums[i] <= nums.length && nums[i] != nums[nums[i] - 1]) {
        swap(nums, i, nums[i] - 1);
      } else {
        i++;
      }
    }
    for (i = 0; i < nums.length; i++) {
      if (nums[i] != (i + 1)) {
        return i + 1;
      }
    }
    return nums.length + 1;
  }

  private void swap(int[] nums, int i, int j) {
    int tmp = nums[i];
    nums[i] = nums[j];
    nums[j] = tmp;
  }
}

Complexity

Time: $O(n)$

Space: $O(1)$
