55. Jump Game

Source link: https://leetcode.com/problems/jump-game/

Design

Solution I

The problem could be simply solve by the greedy method. Each element denotes the farthest index we can reach. If (i + nums[i] > max), it can reach further index, leading to the update of max. If (i + nums[i] < max), max unchanges. While max >= nums.length - 1, we can reach the last index.

Solution II

The backtracking method could be applied to this problem also. If it wants to reach the last index, it needs to check if it can reach the last index by (last_index_value - 1) first. If not, check the next index with (last_index_value - 2). If yes, reaching (last_index_value - 1) means it can reach the last index. Repeat the process. In the end, check if it reaches the first index (index = 0).

Implementation

Solution I

class Solution &#123;
  public boolean canJump(int[] nums) &#123;
    if (nums.length == 1) return true;
    int max = nums[0];
    for (int i = 0; i <= max; i++) &#123;
      if (max >= nums.length - 1) return true;
      max = Math.max(max, i + nums[i]);
    &#125;
    return false;
  &#125;

Solution II

class Solution &#123;
  public boolean canJump(int[] nums) &#123;
    int prev = nums.length - 1; // last index
    for (int i = nums.length - 1; i >= 0; i--) &#123;
      if (i + nums[i] >= prev) &#123;
        prev = i;
      &#125;
    &#125;
    return prev == 0;
  &#125;
&#125;
