LeetCode 744. Find Smallest Letter Greater Than Target

Problem

Source link: Find Smallest Letter Greater Than Target

Given a list of sorted characters letters containing only lowercase letters, and given a target letter target, find the smallest element in the list that is larger than the given target.

Letters also wrap around. For example, if the target is target = 'z' and letters = ['a', 'b'], the answer is 'a'.

Examples:

Input:
letters = ["c", "f", "j"]
target = "a"
Output: "c"

Input:
letters = ["c", "f", "j"]
target = "c"
Output: "f"

Input:
letters = ["c", "f", "j"]
target = "d"
Output: "f"

Input:
letters = ["c", "f", "j"]
target = "g"
Output: "j"

Input:
letters = ["c", "f", "j"]
target = "j"
Output: "c"

Input:
letters = ["c", "f", "j"]
target = "k"
Output: "c"

Note:

  1. letters has a length in range [2, 10000].
  2. letters consists of lowercase letters, and contains at least 2 unique letters.
  3. target is a lowercase letter.

Design

The problem can be solved by the modified binary search pattern. By breaking down the statement, we could found 5 types of situations in this problem:

  1. The target letter is greater than the last element.
  2. The target letter is smaller than the first element.
  3. The target letter is equal to the middle element.
  4. The target letter is smaller than the middle element.
  5. The target letter is greater than the middle element.

First two situations are easy to get the solution, as the problem stated, just simply return the first element of the array.

For the last three situations, we can use binary search to get the required element. One thing to note here is when the target is equal to the middle element, we should eliminate the left part of the array and carry on searching as we aim to find the element right to the target. The process is shown below:

As shown in the figure, we return the element where the start pointer is. However, there’s an edge case should be handled. If the start pointer end up pointing to indexletters.length, then we should return the first element. This can be done by index start % letters.length.

Implementation

class Solution &#123;
  public char nextGreatestLetter(char[] letters, char target) &#123;
    if (target > letters[letters.length - 1] || target < letters[0]) &#123;
      return letters[0];
    &#125;

    int start = 0, end = letters.length - 1;
    while (start <= end) &#123;
      int mid = start + (end - start) / 2;
      if (target >= letters[mid]) &#123;
        start = mid + 1;
      &#125; else &#123;
        end = mid - 1;
      &#125;
    &#125;

    return letters[start % letters.length];
  &#125;
