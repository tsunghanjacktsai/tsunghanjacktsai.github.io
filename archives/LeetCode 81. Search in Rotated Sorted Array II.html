LeetCode 81. Search in Rotated Sorted Array II

Problem

Source link: Search in Rotated Sorted Array II

Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand.

(i.e., [0,0,1,2,2,5,6] might become [2,5,6,0,0,1,2]).

You are given a target value to search. If found in the array return true, otherwise return false.

Example 1:

Input: nums = [2,5,6,0,0,1,2], target = 0
Output: true

Example 2:

Input: nums = [2,5,6,0,0,1,2], target = 3
Output: false

Follow up:

  • This is a follow up problem to Search in Rotated Sorted Array, where nums may contain duplicates.
  • Would this affect the run-time complexity? How and why?

Design

This is an advanced problem of Search in Rotated Sorted Array , which contains duplicates in a rotated sorted array. The key to solve this problem is to handle the situation while both left and right pointers are pointing to the same value as the middle point. Here’s how algorithm process looks like:

  1. Separate the array into two halves by left and right pointers.
  2. Identify which part of the array is sorted.
  3. Determine the target is inside the sorted part or not.
  4. If the middle point is equal to both elements where left and right pointers at. Move either left or right pointer one step forward or backward.
  5. Back to 2nd step until the target is found.

Implementation

class Solution &#123;
  public boolean search(int[] nums, int target) &#123;
    if (nums == null || nums.length == 0) &#123;
      return false;
    &#125;

    int start = 0, end = nums.length - 1;
    while (start <= end) &#123;
      int mid = start + (end - start) / 2;

      if (nums[mid] == target) &#123;
        return true;
      &#125;

      if (nums[start] == nums[mid] && nums[end] == nums[mid]) &#123;
        start++;
      &#125; else if (nums[start] <= nums[mid]) &#123;
        if (target >= nums[start] && target < nums[mid]) &#123;
          end = mid - 1;
        &#125; else &#123;
          start = mid + 1;
        &#125;
      &#125; else if (nums[end] >= nums[mid]) &#123;
        if (target > nums[mid] && target <= nums[end]) &#123;
          start = mid + 1;
        &#125; else &#123;
          end = mid - 1;
        &#125;
      &#125;
    &#125;

    return false;
  &#125;
