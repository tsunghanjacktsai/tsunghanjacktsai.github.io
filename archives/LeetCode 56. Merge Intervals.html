LeetCode 56. Merge Intervals

Problem

Source link: Merge Intervals

Given a collection of intervals, merge all overlapping intervals.

Example 1:

Input: intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]
Explanation: Since intervals [1,3] and [2,6] overlaps, merge them into [1,6].

Example 2:

Input: intervals = [[1,4],[4,5]]
Output: [[1,5]]
Explanation: Intervals [1,4] and [4,5] are considered overlapping.

Constraints:

  • intervals[i][0] <= intervals[i][1]

Design

  1. Sort intervals by their starting value.
  2. Traverse through intervals, if the last value of the previous interval is smaller than the first value of the current interval(e.g. [8, 10] [15, 18] → 15 > 10), which means there is no overlap between them. Add the current interval into the result list.
  3. Otherwise, update the result interval by getting the larger one between ending point of the previous array and ending point of the current array (e.g. [1, 3] [2, 6] → overlap and 6 > 3 → [1, 6]).

Implementation

class Solution {
  public int[][] merge(int[][] intervals) {
    if (intervals == null || intervals.length < 2) {
      return intervals;
    }

    List<int[]> mergedIntervals = new ArrayList<>();
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    int start = intervals[0][0];
    int end = intervals[0][1];

    for (int[] interval : intervals) {
      if (interval[0] <= end) {
        end = Math.max(interval[1], end);
      } else {
        mergedIntervals.add(new int[] {start, end});
        start = interval[0];
        end = interval[1];
      }
    }

    mergedIntervals.add(new int[]{start, end});

    return mergedIntervals.toArray(new int[0][]);
  }
}

Complexity

Time: $O(nlogn)$

Space: $O(n)$
