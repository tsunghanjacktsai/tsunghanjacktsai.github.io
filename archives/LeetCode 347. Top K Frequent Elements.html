LeetCode 347. Top K Frequent Elements

Problem

Source link: Top K Frequent Elements

Given a non-empty array of integers, return the k\ most frequent elements.

Example 1:

Input: nums = [1,1,1,2,2,3], k = 2
Output: [1,2]

Example 2:

Input: nums = [1], k = 1
Output: [1]

Note:

  • You may assume k is always valid, 1 ≤ k ≤ number of unique elements.
  • Your algorithm’s time complexity must be better than O(n log n), where n is the array’s size.
  • It’s guaranteed that the answer is unique, in other words the set of the top k frequent elements is unique.
  • You can return the answer in any order.

Design

Let’s break down the problem step by step. The problem is consisted with two sub-problems here:

  1. Calculate the frequency of each element.
  2. Retrieve the K most frequent ones.

For the first sub-problem, we can utilize a HashMap to handle the elements with their frequency.

After, we can then apply the top K elements pattern as above. However, we could do some variations here. The min heap created here should sort elements based on their frequencies, in other words, it needs to compare each value of the HashMap.

Now we could start to define our steps to solve the problem:

  1. Store the array elements into a HashMap with their frequencies.
  2. Create a min heap which sort the map entries based on their values (element frequency).
  3. Traverse and insert the array elements into the heap. While the size of the heap is larger than k, pop out the root element as we only need to maintain a k size min heap to reach the required result.

Take a look at the following visualization to understand the process:

As the figure shown, since we only need the most two frequent elements, if the size of the heap is larger than 2, its root element will be the 3rd most frequent element, which can be extracted.

Implementation

class Solution &#123;
  public int[] topKFrequent(int[] nums, int k) &#123;
    // Result array
    int[] result = new int[k];

    // Map for the frequencies of each num
    Map<Integer, Integer> numFreq = new HashMap<>();
    // Record the num frequencies
    for (int num : nums) &#123;
      numFreq.put(num, numFreq.getOrDefault(num, 0) + 1);
    &#125;

        // Min heap    
    PriorityQueue<Map.Entry<Integer, Integer>> minHeap = new PriorityQueue<>((a, b) -> a.getValue() - b.getValue());
    // Traverse the frequency map and maintain a k size min heap
    for (Map.Entry<Integer, Integer> entry : numFreq.entrySet()) &#123;
      minHeap.add(entry);
      if (minHeap.size() > k) &#123;
        minHeap.poll();
      &#125;
    &#125;

    // Get the result elements
    for (int i = 0; i < k; i++) &#123;
      result[i] = minHeap.poll().getKey();
    &#125;

    return result;
  &#125;
